 HOW TO USE GITHUB WITHOUT GETTING GIT :)

  by David T. Sanders

This guide shows you how to use JupyterLabs to access the power of github without installing Git. If you’re new to Git and/or GitHub, consider taking one of these courses.

REQUIREMENTS:

  • a modern browser
  • internet access
  • ability to navigate websites
  • a free github.com account

GOALS OF THE GUIDE:

  • Clone an existing GitHub repository into a directory in JupyterLab
  • Modify, stage, and commit some files in the directory to git
  • Push JupyterLab directory into a new GitHub repository

We’ll clone an existing repository from GitHub into our “local” environment, change files, commit them ‘locally”, then push our files to a GitHub repository. Without installing git.

We’ll use a web-based development environment (JupyterLabs) and the command line interface (CLI) to interact  with GitHub. You won’t have to install or pay for anything to complete the tasks in this guide.

WHAT WE NEED TO KNOW:

Jupyterlabs

A free website that allows us to interact with github, and many other programming environments, without logging in or downloading any software. It will act as your local computer when using GitHub. You will need a free GitHub account.

JupyterLabs runs inside of Binder, a behind-the-scenes container that presents JupyterLabs to us through the web. You’ll see it briefly as JupyterLabs opens.

CLI (Command Line Interface)

CLI’s are another way to tell the computer what you want it to do. They’re an alternative to a point-and-click Graphical User Interface (GUI). To use the CLI in JupyterLab, you need to have a basic understanding of two things, Commands and the Terminal. Commands are typed into the terminal.

  • Commands
    Commands are a series of english statements that humans use to communicate with the computer. If we use the correct commands, Jupyter and GitHub will interact to version our files.

  • Terminal

The Terminal is a window inside JupyterLabs that has a blinking cursor. It waits for your commands. You type commands into the Terminal to initiate git actions.

Example of typical CLI commands you will type or copy/paste into the Terminal:

 git clone repository_url                  # repo you want to download from github

 git remote -v                                # see which remote servers you have configured

 git add file_name                        # file you want to upload to a repo

 git commit -m “my first commit”        # commits changes locally (JupyterLabs)

 git pull                                        # pulls changes from repository

 git push                                # uploads committed file to repository

GITHUB

A free document storage and versioning web site that is used by many businesses and individuals to keep track of changes to documents. There are lots of tutorials and code examples. Learning github is worth the effort. Get you free account here:

https://github.com/join?ref_cta=Sign+up&ref_loc=header+logged+out&ref_page=%2F&source=header-home 

PUTTING IT ALL TOGETHER:

In the Jupyterlab website,  you’ll open a Terminal session and type a few short commands. You’ll then have access to GitHub repositories. You can add, commit, clone, fork, push, pull, and more for robust version control of files. Without paying a dime or installing Git.

As of this writing (Feb 2021), the Jupyter-git extension that provides a graphical user interface between Jupyter and GitHub has been disabled due to lags . Fortunately, we’re using the CLI to interact with JupyterLabs and GitHub so this won’t slow our progress.

Connect to GitHub from Jupyter:

Once you’re in a Terminal session inside of JupyterLabs, you can connect to GitHub using git commands. More tutorials and great resources:

Let’s start by cloning an existing GitHub repo into our JupyterLabs instance.  Cloning a repo will also set git to point to that remote repo. When you push files to GitHub, they will go to that repo, unless you change the remote first.

In this guide, we will connect to a GitHub repository called github/training-kit. It has lots of great examples of Git CLI instructions that you can use in JupyterLab

https://github.com/github/training-kit.git 

https://github.com/github/training-kit/tree/master/git-guides

As a recap… We will clone a GitHub repository into our JupyterLabs instance using commands we type into the Terminal.

GITHUB - CLI COMMANDS (enter into the Terminal)

##############################################################################

####    Any line that starts with a hashtag is a comment. All other lines should be entered into ####   the Terminal one at a time in order              

##############################################################################

##############################################################################
